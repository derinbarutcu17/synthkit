# Contributing

SynthKit is optimized for boring reliability.

## Ground rules

- Keep changes contract-driven.
- Add tests when you fix a bug.
- Do not let clients invent private payloads.
- Prefer honest failure modes over fake confidence.

## Workflow

1. Make a small change.
2. Run the relevant tests.
3. Update docs if the contract changed.
4. Prefer local-first behavior over external dependencies.
